# 公開中継サーバー セットアップガイド

会場の採点システム（server.js）から確定得点だけを受け取り、インターネット経由の観覧者にリアルタイムで配信する「中継サーバー」です。観覧者側からは閲覧専用で、得点の入力・編集は一切できません。

---

## 1. デプロイ先を決める（おすすめ: Render.com）

※以前はGlitchを案内していましたが、Glitchは2025年7月にアプリホスティングを終了しているため使えません。代わりに **Render.com** を使います。無料プランがあり、クレジットカード登録不要、Dockerfileなどの専門知識も不要です（ただしGitHubリポジトリへのアップロードが必要です）。

### 事前準備: GitHubにこのフォルダをアップロードする

Gitコマンドを使わなくても、ブラウザだけで完了できます。

1. https://github.com にアクセスし、無料アカウントを作成（未登録の場合）。
2. 右上の「+」→「New repository」で新しいリポジトリを作成（例: `gym-public-relay`）。「Public」のままでOK。
3. 作成後の画面で「uploading an existing file」というリンクをクリックし、この`cloud-relay`フォルダの中身（`server.js`・`package.json`・`public`フォルダ一式）をドラッグ&ドロップでアップロードして「Commit changes」。

### Renderでのデプロイ手順

1. https://render.com にアクセスし、GitHubアカウントでログイン（無料）。
2. ダッシュボードで「New +」→「Web Service」を選択。
3. 先ほど作成したGitHubリポジトリ（`gym-public-relay`）を選択して連携。
4. 設定項目:
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. 「Environment」タブで環境変数を追加:
   ```
   RELAY_KEY = 好きな合言葉（例: gym2026test）
   ```
   ※この値は、あとで会場側の`admin.html`にも同じ文字列を入力します。第三者に知られないようにしてください。
6. 「Create Web Service」をクリックすると数分でビルド・起動し、`https://（サービス名).onrender.com` のようなURLが発行されます。これが公開URLです。

**注意（無料プランの制約）**: 15分間アクセスが無いとスリープし、次のアクセス時に再起動で1分ほどかかります。観覧者の最初のアクセスが少し遅れることがありますが、テスト運用では実害は小さいはずです。本番で常時スリープなしにしたい場合は有料プラン（Starter以上）への切り替えで解消できます。

### 動作確認しながら短期間だけ使いたい場合: Railway

１回のイベント運用だけなら、GitHub連携が同じくらい簡単で初回登録時に無料クレジットが付与される **Railway**（https://railway.app）も候補です。ただしクレジットを使い切ると有料（月額制）に切り替わる点に注意してください。

---

## 2. 公開URLを会場側の管理画面に設定する

会場の`admin.html`（更新版）を開き、「🌐 公開ページ（クラウド中継）連携」セクションに以下を入力:

- **中継サーバーURL**: `https://（サービス名).onrender.com` のような、上で発行されたURL
- **合言葉（Key）**: `.env`に設定した`RELAY_KEY`と同じ文字列

保存すると、以後「得点確定」や「場内表示への反映」のたびに、自動でこの中継サーバーへデータが転送されます。

---

## 3. 観覧者に共有するURL

- 記録一覧（公開版）: `https://（サービス名).onrender.com/scoreboard.html`
- 場内表示と同じ画面（公開版）: `https://（サービス名).onrender.com/display.html`

QRコードにして配布すると便利です（admin.htmlのQR生成機能で、IPアドレス欄の代わりにこのURLを使う形でも応用できます）。

---

## 4. 動作確認のしかた

1. 中継サーバーのURL末尾に `/health` を付けてブラウザでアクセス（例: `https://xxxx.onrender.com/health`）。`{"status":"ok",...}` が表示されれば起動成功。
2. 会場側のadmin.htmlで「テスト送信」ボタンを押し、観覧用の `scoreboard.html` を別のスマホ等で開いてデータが反映されるか確認。

## 5. 仕組みの補足

- `/push` というエンドポイントだけが書き込み可能で、`x-relay-key` ヘッダーが一致しないと拒否されます。
- 観覧者が接続するWebSocketは送信専用（受信側）として動いており、観覧者のブラウザから何を送っても無視されます。得点の改ざんはできない設計です。
